3. 提供移动端和PC端两种界面
4. 可嵌入到现有RM系统

### 核心算法

#### 算法1: 价格→保证金比例
```javascript
function calculateMarginRatio(ftp, targetPrice) {
  const priceDiff = targetPrice - ftp;
  const ratio = priceDiff / targetPrice;
  return Math.ceil(ratio * 100) / 100; // 向上取整
}
```

#### 算法2: 保证金比例→价格
```javascript
function calculateMinPrice(ftp, targetRatio) {
  const basePrice = (ftp * targetRatio) / (1 - targetRatio);
  const roundedBase = Math.floor(basePrice * 100) / 100; // 向下取整
  return roundedBase + ftp;
}
```

## 使用场景

### 场景1: 客户现场询价（移动端）
**用户**: RM客户经理
**场景**: 外出拜访客户，客户询问承兑业务价格
**操作**:
1. 打开移动端计算器
2. 输入客户期望价格
3. 3秒内得出保证金比例
4. 现场给出专业报价

### 场景2: 方案对比分析（PC端）
**用户**: RM客户经理
**场景**: 办公室为客户设计多个承兑方案
**操作**:
1. 打开PC端计算器
2. 计算不同价格对应的保证金比例
3. 或计算不同保证金比例对应的价格
4. 导出结果发送给客户

### 场景3: FTP调整（配置管理）
**用户**: 风控部门/管理员
**场景**: 市场利率变化，需要调整FTP
**操作**:
1. 打开FTP配置管理页面
2. 输入新的FTP值和生效日期
3. 保存配置
4. 所有计算器自动使用新FTP

## 知识沉淀

### 业务知识
1. **FTP概念**: 资金转移定价，银行内部资金成本基准
2. **保证金比例**: 客户需缴纳的保证金占承兑金额的比例
3. **取整规则**: 保证金比例向上取整，价格向下取整，对银行有利
4. **合理范围**: 保证金比例通常30%-90%，FTP通常1.5%-3.5%

### 技术知识
