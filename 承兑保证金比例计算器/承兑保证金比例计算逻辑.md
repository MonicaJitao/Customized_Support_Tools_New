# 承兑保证金比例计算逻辑

## 核心参数

- **FTP**: 资金转移定价（Funds Transfer Pricing），基准利率
- **目标锚定价格**: 客户期望的承兑价格
- **目标保证金比例**: 客户期望的保证金占比
- **0手续费保证金比例**: 在不收取手续费情况下的保证金比例
- **对应最低价格**: 达到目标保证金比例所需的最低承兑价格

## 计算公式

### 公式1: 通过目标锚定价格计算0手续费保证金比例

**输入**:
- FTP (例如: 2.13)
- 目标锚定价格 (例如: 9.99)

**计算过程**:
1. 计算价格差: `价格差 = 目标锚定价格 - FTP`
2. 计算保证金比例: `保证金比例 = 价格差 / 目标锚定价格`
3. 向上取整到小数点后2位

**Excel公式**: `=ROUNDUP((B4-B1)/B4, 2)`

**JavaScript实现**:
```javascript
function calculateMarginRatio(ftp, targetPrice) {
  const priceDiff = targetPrice - ftp;
  const ratio = priceDiff / targetPrice;
  return Math.ceil(ratio * 100) / 100; // 向上取整到2位小数
}
```

**示例**:
- FTP = 2.13
- 目标锚定价格 = 9.99
- 价格差 = 9.99 - 2.13 = 7.86
- 保证金比例 = 7.86 / 9.99 = 0.7868...
- 向上取整 = 0.79 (79%)

### 公式2: 通过目标保证金比例计算对应最低价格

**输入**:
- FTP (例如: 2.13)
- 目标保证金比例 (例如: 0.55 即 55%)

**计算过程**:
1. 计算基础价格: `基础价格 = (FTP × 目标保证金比例) / (1 - 目标保证金比例)`
2. 向下取整到小数点后2位
3. 加上FTP得到最终价格: `最低价格 = 基础价格 + FTP`

**Excel公式**: `=ROUNDDOWN((B1*B8)/(1-B8), 2) + B1`

**JavaScript实现**:
```javascript
function calculateMinPrice(ftp, targetRatio) {
  const basePrice = (ftp * targetRatio) / (1 - targetRatio);
  const roundedBase = Math.floor(basePrice * 100) / 100; // 向下取整到2位小数
  return roundedBase + ftp;
}
```

**示例**:
- FTP = 2.13
- 目标保证金比例 = 0.55 (55%)
- 基础价格 = (2.13 × 0.55) / (1 - 0.55) = 1.1715 / 0.45 = 2.603...
- 向下取整 = 2.60
- 最低价格 = 2.60 + 2.13 = 4.73

## 业务逻辑说明

### 计算模式1: 价格反推保证金比例
- **使用场景**: 客户已有目标承兑价格，需要知道对应的保证金比例
- **输入**: FTP + 目标锚定价格
- **输出**: 0手续费保证金比例
- **业务含义**: 在给定价格下，银行需要收取多少比例的保证金才能覆盖成本

### 计算模式2: 保证金比例反推价格
- **使用场景**: 客户有保证金比例限制，需要知道最低承兑价格
- **输入**: FTP + 目标保证金比例
- **输出**: 对应最低价格
- **业务含义**: 在给定保证金比例下，承兑价格至少要达到多少才能覆盖成本

## 数据验证规则

1. **FTP**: 必须 > 0，通常在 1.5 - 3.5 之间
2. **目标锚定价格**: 必须 > FTP
3. **目标保证金比例**: 必须在 0 - 1 之间（0% - 100%）
4. **计算结果**: 保证金比例应在合理范围内（通常 30% - 90%）

## 取整规则

- **保证金比例计算**: 向上取整（ROUNDUP）- 对银行有利，确保覆盖成本
- **价格计算**: 向下取整（ROUNDDOWN）- 对银行有利，确保保证金充足
