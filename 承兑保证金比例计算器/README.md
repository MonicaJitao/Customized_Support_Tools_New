# 承兑保证金比例计算器

## 项目概述

本项目是一个用于计算承兑业务保证金比例的Web应用，支持移动端和PC端两种界面，可嵌入到RM展业系统中使用。采用白底蓝色系简洁风格，支持权限管理。

## 功能特性

### 1. 移动端计算器
- **界面风格**: 白底蓝色系，简洁平铺式设计
- **响应式设计**: 适配各种移动设备屏幕
- **双模式计算**:
  - 模式1: 通过目标锚定价格计算0手续费保证金比例
  - 模式2: 通过目标保证金比例计算对应最低价格
- **简洁结果**: 只显示计算结果，无冗余信息

### 2. PC端计算器
- **界面风格**: 白底蓝色系，专业商务风格
- **左右分栏布局**: 左侧输入参数，右侧显示结果
- **权限管理**: 支持管理员和普通用户两种角色
- **配置中心**: 管理员可访问FTP配置管理（弹窗形式）
- **FTP保护**: 普通用户无法修改FTP值

### 3. FTP配置管理
- **独立配置页面**: 专门的FTP配置管理界面
- **历史记录**: 保存所有FTP配置历史

- **数据持久化**: 使用localStorage存储配置数据
- **配置导出**: 支持导出配置为JSON文件

- **普通用户权限**: 只能使用计算功能，FTP输入框禁用
- **灵活集成**: 支持与RM系统权限体系对接

## 核心计算逻辑

### 公式1: 价格 → 保证金比例

**输入**: 应用FTP + 目标锚定价格
**输出**: 0手续费保证金比例

```javascript
保证金比例 = ROUNDUP((目标锚定价格 - FTP) / 目标锚定价格, 2)
```

**示例**:
- FTP = 2.13%
- 目标锚定价格 = 9.99%
- 保证金比例 = 79%

### 公式2: 保证金比例 → 价格

**输入**: 应用FTP + 目标保证金比例
**输出**: 对应最低价格

```javascript
基础价格 = ROUNDDOWN((FTP × 目标保证金比例) / (1 - 目标保证金比例), 2)
最低价格 = 基础价格 + FTP
```

**示例**:
- FTP = 2.13%
- 目标保证金比例 = 55%
- 最低价格 = 4.73%

## 使用说明

### 移动端使用

1. 打开 `承兑保证金计算器_移动端.html`
2. 选择计算模式（价格→保证金 或 保证金→价格）
3. 输入应用FTP和目标参数
4. 点击计算按钮查看结果

### PC端使用

#### 管理员用户
1. 打开 `承兑保证金计算器_PC端.html`
2. 右上角显示"⚙️ 配置中心"按钮
3. 可点击进入FTP配置管理
4. 可临时修改FTP值进行计算
5. 在左侧选择计算模式并输入参数
6. 点击计算按钮，右侧显示结果

#### 普通用户
1. 打开 `承兑保证金计算器_PC端.html`
2. 右上角无"配置中心"按钮
3. FTP输入框禁用（灰色显示）
4. 使用系统配置的FTP进行计算
5. 在左侧选择计算模式并输入参数
6. 点击计算按钮，右侧显示结果

### 权限设置

#### 方法1: JavaScript调用（推荐）
```javascript
// 设置为管理员
setUserRole('admin');

// 设置为普通用户
setUserRole('user');
```

#### 方法2: 集成到RM系统
```javascript
// 根据RM系统用户角色设置
const rmUserRole = getCurrentUserRole();

if (rmUserRole === '管理员' || rmUserRole === '风控人员') {
    localStorage.setItem('userRole', 'admin');
} else {
    localStorage.setItem('userRole', 'user');
}
```

详细权限管理说明请查看 `权限管理说明.md`

### FTP配置管理

1. 管理员在PC端计算器点击"配置中心"
2. 或直接打开 `FTP配置管理_PC端.html`
3. 输入应用FTP值、生效日期和备注说明
4. 点击"保存配置"
5. 历史记录会自动保存，可随时查看和应用历史配置

## 数据验证规则

1. **应用FTP值**: 必须 > 0，建议范围 1.5% - 3.5%
2. **目标锚定价格**: 必须 > FTP
3. **目标保证金比例**: 必须在 0% - 100% 之间
4. **计算结果**: 保证金比例通常在 30% - 90% 之间

## 取整规则说明

- **保证金比例计算**: 向上取整（ROUNDUP）- 确保银行风险覆盖
- **价格计算**: 向下取整（ROUNDDOWN）- 确保保证金充足

这种取整策略对银行有利，符合风险控制原则。

## 业务场景

### 场景1: 客户询价

### 场景2: 保证金限制
客户最多只能提供55%的保证金，RM使用计算器计算出最低承兑价格为4.73%。

### 场景4: 权限控制
普通RM只能使用计算功能，无法修改FTP；管理员可以配置FTP并管理历史记录。

## 常见问题

**Q: 为什么目标价格必须大于FTP？**
A: FTP是资金成本，承兑价格必须覆盖成本才能盈利。

**Q: 保证金比例为什么向上取整？**
A: 向上取整确保银行收取足够的保证金覆盖风险。

**Q: FTP配置在哪里存储？**
A: 使用浏览器localStorage存储，数据保存在本地。

**Q: 如何设置用户权限？**
A: 通过 `setUserRole('admin')` 或 `setUserRole('user')` 设置，详见权限管理说明。

**Q: 普通用户能修改FTP吗？**
A: 不能。普通用户的FTP输入框是禁用状态，只能使用系统配置的FTP。

**Q: 管理员临时修改的FTP会保存吗？**
A: 不会。只有通过"配置中心"修改的FTP才会持久化保存。

**Q: 如何清除历史数据？**
A: 在浏览器开发者工具中清除localStorage，或在FTP配置页面逐条删除。
