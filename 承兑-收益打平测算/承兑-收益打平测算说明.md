## 5. 金融口径说明

### 5.1 FTP 是什么

FTP（Funds Transfer Pricing，资金转移定价）是商业银行内部司库与业务经营单位之间，按照一定规则对资金进行有偿内部转移的定价机制，用于核算业务的资金成本或资金收益。

在 FTP 口径下：

- 对资产业务，经营单位通常按 FTP 向内部资金中心“买资金”，因此 FTP 表现为资金成本。
- 对负债业务，经营单位通常按 FTP 向内部资金中心“卖资金”，因此 FTP 表现为资金收益。

### 5.2 本工作簿中的“保证金FTP收入/支出”应如何理解

从字段命名和公式结构看，本工作簿把保证金视为一类可沉淀资金，并按不同客户付息方式拆成三种口径：

- 活期
- 3 个月定期
- 6 个月定期

因此：

- `保证金FTP收入` 可理解为该类保证金资金在内部资金转移定价体系下形成的内部收益率。
- `保证金FTP活期支出 / 3个月定期支出 / 6个月定期支出` 可理解为对客户端负债成本或付息成本的不同口径。
- 二者相减后的 `B5:B7`，就是保证金在不同付息方式下的净 FTP 收益率。

这与 FTP 的通用定义一致：对负债类资金，经营单位获得 FTP 收益，再扣除对客户的实际付息支出，形成净收益。

### 5.3 战略业务成本、信贷FTP、风险成本

这些概念在本工作簿中按内部模型参数使用：

- `战略业务成本`：按工作簿口径，是承兑业务中需要覆盖的一项成本率，直接进入核心打平公式。
- `信贷FTP`：按工作簿口径，是与信贷资金占用相关的 FTP 成本率。
- `优化后风险成本`：按工作簿口径，是模型中纳入的一项风险成本率。

这三个字段都直接进入核心测算公式，因此在系统实现时应作为明确定义的参数来源，不应在计算代码中写死业务含义后再反推公式。

## 7. 右侧情景测算区与图表逻辑

### 7.1 情景遍历轴

`H2:H102` 为保证金比例情景轴：

- `H2 = 0`
- `H3 = 0.01`
- ...
- `H102 = 1`

即从 `0%` 到 `100%`，步长为 `1%`，共 `101` 个点。

### 7.2 原始测算列 `J`

`J2:J102` 是对不同保证金比例情景进行遍历后的原始测算结果。

以 `J2` 为例，工作簿中的精确公式为：

```excel
=((B$15*B$5)-(B$14*(B$8-B$9))-(B$14*B$17))/((H2*B$5)-((1-B$22)*(B$8-B$9))-((1-H2)*B$17)-((B$12-H2)*(B$13-B$8-B$17)*B$16))
```

`J3:J102` 使用同结构共享公式，仅把 `H2` 替换为对应行的 `Hn`。

可概括写成：

```excel
Jn = ((B$15*B$5)-(B$14*(B$8-B$9))-(B$14*B$17))
     /
     ((Hn*B$5)-((1-B$22)*(B$8-B$9))-((1-Hn)*B$17)-((B$12-Hn)*(B$13-B$8-B$17)*B$16))
```

注意：

- `J` 列固定使用的是 `B5`，即“活期”保证金净 FTP 收益率。
- 因此右侧图表实际上展示的是“活期口径下，保证金比例变化对未来三个月开票金额的影响”。

### 7.3 图表展示列 `I`

`I2:I102` 不是直接测算值，而是对 `J` 列做了一层区间过滤：

`I2`：

```excel
=IF(OR(J2<0,J2>2000),,J2)
```

`I3:I102` 同理。

含义：

- 若 `Jn < 0`，则 `In` 置空
- 若 `Jn > 2000`，则 `In` 置空
- 只有 `0 <= Jn <= 2000` 时，`In = Jn`

因此，图表展示的不是全部原始测算值，而是经过“合理区间过滤”后的结果。

补充说明：

- 从公式语义看，超出区间的点应返回空白。
- 从工作簿当前图表缓存值看，部分空白点在缓存层被保存为 `0`。
- 因此，如果后续系统要复刻 Excel 图形效果，建议同时验证“空白点显示策略”和“图表库对空值的默认处理方式”。

### 7.4 图表对象定义

工作簿中的“图”不是静态图片，而是 Excel 内嵌折线图对象。

图表定义如下：

- 图表类型：折线图（`lineChart`）
- 图表标题：`未来三个月开票金额`
- 系列名称：引用 `Sheet1!$I$1`
- 横轴数据源：`Sheet1!$H$2:$H$102`
- 纵轴数据源：`Sheet1!$I$2:$I$102`

也就是说，图表展示的是：

- 横轴：保证金比例
- 纵轴：未来三个月开票金额

### 7.5 图表数据范围的业务含义

按当前工作簿缓存结果，`J` 列的取值区间可分为三段：

- `0%` 到 `30%`：原始测算值为负数，因此 `I` 列按公式应置空
- `31%` 到 `39%`：原始测算值大于 `2000`，因此 `I` 列按公式应置空
- `40%` 到 `100%`：原始测算值位于 `0` 到 `2000` 之间，进入图表有效区间

| 保证金比例 | 单元格 | 结果（万） | 说明 |
