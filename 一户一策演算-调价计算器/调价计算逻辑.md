# 一户一策演算-调价计算逻辑（修订版）

## 核心参数

### 可配置参数（需要配置页面）

| 参数名称 | 默认值 | 说明 | 单位 |
|---------|--------|------|------|
| 业务成本 (fxcb) | 3.54 | 资金成本 | % |
| 计提利润点 | 101 | 利润点数 | bp (基点) |
| 520客户兜底价格 | 4.55 | 520客户固定兜底价格 | % |

**注意**:
- 1bp = 0.01%，因此 101bp = 1.01%
- 非520客户兜底价格 = max(业务成本 + 风险成本, 4.55)

### 用户输入参数

| 参数名称 | 说明 | 单位 |
|---------|------|------|
| 520客户 | Y/N | - |
| 当前价格 | 客户当前贷款价格 | % |
| 当前风险成本 | 风险成本 | % |
| 近3个月长周期余额 | 近3个月的平均余额 | 万元 |
| 目标价格 | 调价后的目标价格 | % |
| 当前可用额度 | 客户当前可用额度 | 万元 |
| 预计用款天数 | 预计使用天数 | 天 |

## 计算逻辑

### 步骤1: 确定兜底价格

根据客户类型确定兜底价格：

```javascript
if (是520客户 === 'Y') {
    兜底价格 = 520客户兜底价格  // 固定值，默认4.55
} else {
    兜底价格 = max(业务成本 + 当前风险成本, 4.55)
    // 例如: max(3.54 + 0.5, 4.55) = 4.55
    // 或: max(3.54 + 1.2, 4.55) = 4.74
}
```

### 步骤2: 计算对应3个月长周期目标余额

核心公式：

```
对应3个月长周期目标余额 = (当前价格 - 兜底价格) × 近3个月长周期余额 / (目标价格 - 兜底价格)
```

**Excel对应**: `=($A$6-$D$4)*$A$8/(G2-$D$4)`

其中：
- 当前价格 - 兜底价格 = 当前利润空间
- 目标价格 - 兜底价格 = 目标利润空间
- 通过利润守恒计算目标余额

### 步骤3: 计算日均需提额余额

```
日均需提额余额 = 对应3个月长周期目标余额 - 近3个月长周期余额
```

**业务含义**:
- 正值表示需要增加余额
- 负值表示可以减少余额

### 步骤4: 计算对应用款天数目标余额

根据预计用款天数计算：

```
对应用款天数目标余额 = (90 / 预计用款天数) × 对应3个月长周期目标余额
```

**Excel对应**: `=90/A18*A12`

**业务含义**:
- 90天为基准周期（3个月）
- 预计用款天数越短，目标余额越高
- 预计用款天数越长，目标余额越低

## 输出结果

### 1. 对应3个月长周期目标余额

**说明**: 在目标价格下，保持相同利润需要的3个月平均余额

**计算**: 基于利润守恒原则

### 2. 日均需提额余额

**说明**: 相对于当前余额，需要增加或减少的余额

**计算**: 目标余额 - 当前余额

### 3. 对应用款天数目标余额

**说明**: 根据预计用款天数，对应的目标余额

**计算**: 按天数比例计算

### 4. 曲线图

**X轴**: 目标价格 (%)
**Y轴**: 对应3个月长周期目标余额 (万元)

**曲线特性**:
- 价格越低，需要的余额越高
- 呈反比关系
- 标记当前点和目标点

## 业务场景

### 场景1: 520客户降价

**输入**:
- 520客户: Y
- 520客户兜底价格: 4.55%
- 当前价格: 7.99%
- 近3个月长周期余额: 205万元
- 目标价格: 6.00%
- 当前可用额度: 180万元
- 预计用款天数: 90天

**计算**:
1. 兜底价格 = 4.55%（520客户固定兜底价格）
2. 对应3个月长周期目标余额 = (7.99 - 4.55) × 205 / (6.00 - 4.55) = 486.55万元
3. 日均需提额余额 = 486.55 - 205 = 281.55万元
4. 对应用款天数目标余额 = (90 / 90) × 486.55 = 486.55万元

### 场景2: 非520客户调价

**输入**:
- 520客户: N
- 业务成本: 3.54%
- 当前风险成本: 0.5%
- 当前价格: 7.99%
- 近3个月长周期余额: 205万元
- 目标价格: 5.00%
- 当前可用额度: 180万元
- 预计用款天数: 60天

**计算**:
1. 兜底价格 = max(3.54 + 0.5, 4.55) = 4.55%
2. 对应3个月长周期目标余额 = (7.99 - 4.55) × 205 / (5.00 - 4.55) = 1,568.67万元
3. 日均需提额余额 = 1,568.67 - 205 = 1,363.67万元
4. 对应用款天数目标余额 = (90 / 60) × 1,568.67 = 2,353.00万元

## 数据验证规则

1. **业务成本**: 必须 > 0
2. **计提利润点**: 必须 > 0
3. **当前风险成本**: 必须 ≥ 0
4. **当前价格**: 必须 > 兜底价格
5. **目标价格**: 必须 > 兜底价格
6. **近3个月长周期余额**: 必须 > 0
7. **当前可用额度**: 必须 > 0
8. **预计用款天数**: 必须 > 0

## 特殊情况处理

### 情况1: 目标价格 ≤ 兜底价格

**提示**: "目标价格不能低于兜底价格"

**原因**: 低于兜底价格会导致亏损

### 情况2: 当前价格 ≤ 兜底价格

**提示**: "当前价格低于兜底价格，业务不合理"

**原因**: 当前业务已经亏损

### 情况3: 目标余额过大

**提示**: "目标余额超出合理范围，请重新评估"

**原因**: 余额过大可能不符合业务实际

## 公式推导

### 核心公式推导

假设：
- P₀ = 当前价格
- P₁ = 目标价格
- Pb = 兜底价格
- A₀ = 近3个月长周期余额
- A₁ = 对应3个月长周期目标余额

**利润守恒原则**:

```
(P₀ - Pb) × A₀ = (P₁ - Pb) × A₁
```

**推导目标余额**:

```
A₁ = (P₀ - Pb) × A₀ / (P₁ - Pb)
```

**业务含义**:
- 分子 `(P₀ - Pb) × A₀` 表示当前的总利润空间
- 分母 `(P₁ - Pb)` 表示目标价格的单位利润空间
- 结果表示在目标价格下需要多少余额才能保持相同的总利润
