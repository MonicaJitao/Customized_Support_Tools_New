# 一户一策演算-调价计算器

## 项目概述

本项目是一个用于计算客户调价后目标额度的Web应用，支持移动端和PC端两种界面，可嵌入到RM展业系统中使用。采用白底蓝色系简洁风格。

## 项目结构

```
一户一策演算-调价计算器/
├── 调价计算器_移动端.html    # 移动端计算器页面
├── 调价计算器_PC端.html      # PC端计算器页面
├── 调价计算逻辑.md           # 计算逻辑详细文档
└── README.md                 # 项目说明文档（本文件）
```

## 功能特性

### 1. 移动端计算器
- **界面风格**: 白底蓝色系，简洁平铺式设计
- **客户类型**: 支持520客户(Y)和非520客户(N)
- **参数配置**: 从localStorage读取配置参数
- **计算功能**: 输入当前价格、风险成本、余额等，计算三个目标结果
- **可视化**: 价格-余额曲线图，标记当前点和目标点，带刻度和图例

### 2. PC端计算器
- **界面风格**: 白底蓝色系，专业商务风格
- **左右分栏布局**: 左侧参数输入，右侧结果展示
- **详细结果**: 显示三个计算结果和所有参数
- **交互式图表**: 更大的曲线图，带网格线、刻度、数据标签和图例
- **权限管理**: 管理员可访问参数配置中心

### 3. 参数配置管理（PC端）
- **配置参数**: 业务成本、计提利润点、非520客户兜底价格
- **历史记录**: 保存所有配置变更历史
- **权限控制**: 仅管理员可访问

## 核心计算逻辑

### 可配置参数（通过配置页面管理）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 业务成本 (fxcb) | 3.54% | 资金成本 |
| 计提利润点 | 101bp | 利润点数，1bp=0.01% |
| 非520客户兜底价格 | 4.55% | 非520客户最低价格 |

### 用户输入参数

| 参数 | 说明 | 单位 |
|------|------|------|
| 520客户 | Y/N | - |
| 当前价格 | 客户当前贷款价格 | % |
| 当前风险成本 | 风险成本 | % |
| 近3个月长周期余额 | 近3个月的平均余额 | 万元 |
| 目标价格 | 调价后的目标价格 | % |
| 当前可用额度 | 客户当前可用额度 | 万元 |
| 预计用款天数 | 预计使用天数 | 天 |

### 计算公式

#### 1. 兜底价格计算

```javascript
if (客户类型 === 'Y') {
    兜底价格 = 业务成本 + 当前风险成本 + (计提利润点 / 100)
    // 例如: 3.54 + 0.5 + 1.01 = 5.05
} else {
    兜底价格 = 非520客户兜底价格
    // 例如: 4.55
}
```

#### 2. 对应3个月长周期目标余额

```
对应3个月长周期目标余额 = (当前价格 - 兜底价格) × 近3个月长周期余额 / (目标价格 - 兜底价格)
```

**示例**:
- 当前价格: 7.99%
- 兜底价格: 5.05%
- 近3个月长周期余额: 205万元
- 目标价格: 6.00%
- 对应3个月长周期目标余额 = (7.99 - 5.05) × 205 / (6.00 - 5.05) = 634.74万元

#### 3. 日均需提额余额

```
日均需提额余额 = 对应3个月长周期目标余额 - 近3个月长周期余额
```

#### 4. 对应用款天数目标余额

```
对应用款天数目标余额 = (预计用款天数 / 90) × 对应3个月长周期目标余额
```

## 使用说明

### 移动端使用

1. 打开 `调价计算器_移动端.html`
2. 选择客户类型（520客户/非520客户）
3. 输入计算参数：
   - 当前价格
   - 当前风险成本
   - 近3个月长周期余额
   - 目标价格
   - 当前可用额度
   - 预计用款天数
4. 点击"计算目标余额"
5. 查看三个结果和曲线图

### PC端使用

1. 打开 `调价计算器_PC端.html`
2. 左侧选择客户类型
3. 输入计算参数
4. 点击"计算目标额度"
5. 右侧查看详细结果和曲线图
6. 管理员可点击右上角"参数配置中心"修改配置

### 参数配置（管理员）

1. 在PC端计算器右上角点击"参数配置中心"
2. 修改业务成本、计提利润点、非520客户兜底价格
3. 设置生效日期和备注
4. 点击"保存配置"
5. 查看历史配置记录

## 业务场景

### 场景1: 520客户降价

**背景**: 520客户当前价格7.99%，近3个月长周期余额205万元，希望降价至6.00%

**操作**:
1. 选择"520客户"
2. 输入当前价格: 7.99
3. 输入当前风险成本: 0.5
4. 输入近3个月长周期余额: 205
5. 输入目标价格: 6.00
6. 输入当前可用额度: 180
7. 输入预计用款天数: 90
8. 计算结果:
   - 对应3个月长周期目标余额: 634.74万元
   - 日均需提额余额: +429.74万元
   - 对应用款天数目标余额: 634.74万元

**业务含义**: 降价需要大幅增加余额以保持利润

### 场景2: 非520客户调价

**背景**: 非520客户当前价格7.99%，近3个月长周期余额205万元，调价至5.00%

**操作**:
1. 选择"非520客户"
2. 输入当前价格: 7.99
3. 输入当前风险成本: 0（非520客户不需要）
4. 输入近3个月长周期余额: 205
5. 输入目标价格: 5.00
6. 输入当前可用额度: 180
7. 输入预计用款天数: 60
8. 计算结果:
   - 对应3个月长周期目标余额: 1,568.67万元
   - 日均需提额余额: +1,363.67万元
   - 对应用款天数目标余额: 1,045.78万元

## 曲线图说明

### 图表元素

- **蓝色曲线**: 价格-余额关系曲线
- **绿色点**: 当前价格和余额
- **红色点**: 目标价格和余额
- **网格线**: 辅助读取数值
- **刻度标签**: X轴显示价格(%)，Y轴显示余额(万元)
- **数据标签**: 在关键点显示具体数值
- **图例**: 标识当前状态和目标状态

### 曲线含义

曲线展示了在不同目标价格下，需要的目标余额。价格越低，需要的余额越高，呈反比关系。

## 数据验证规则

1. **业务成本**: 必须 > 0
2. **计提利润点**: 必须 > 0
3. **当前风险成本**: 必须 ≥ 0
4. **当前价格**: 必须 > 兜底价格
5. **目标价格**: 必须 > 兜底价格
6. **近3个月长周期余额**: 必须 > 0
7. **当前可用额度**: 必须 > 0
8. **预计用款天数**: 必须 > 0

## 常见问题

**Q: 为什么目标价格必须大于兜底价格？**
A: 兜底价格是保本价格，低于此价格会导致亏损。

**Q: 520客户和非520客户有什么区别？**
A: 520客户的兜底价格根据业务成本+风险成本+利润点计算，非520客户使用固定兜底价格。

**Q: 日均需提额余额为负数是什么意思？**
A: 表示可以减少余额，即目标价格高于当前价格时，可以减少余额。

**Q: 曲线图为什么是反比关系？**
A: 根据公式，价格越低，需要的余额越高，以保持相同的利润空间。

**Q: 如何访问参数配置中心？**
A: 需要管理员权限，在浏览器控制台执行 `localStorage.setItem('userRole', 'admin')`，然后刷新页面，点击右上角"参数配置中心"按钮。

**Q: 配置参数保存在哪里？**
A: 保存在浏览器的localStorage中，清除浏览器数据会丢失配置。
