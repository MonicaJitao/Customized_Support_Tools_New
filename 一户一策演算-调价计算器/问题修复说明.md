### 5. ✅ 计算结果与Excel一致性修复

**问题分析**:
根据截图对比，主要差异来源于兜底价格计算逻辑不同。

**Excel逻辑**:
- 520客户：兜底价格 = 4.55%（固定）
- 非520客户：兜底价格 = max(3.54 + 风险成本, 4.55)

**原计算器逻辑**（错误）:
- 520客户：兜底价格 = 3.54 + 风险成本 + 1.01
- 非520客户：兜底价格 = 4.55%（固定）

**修复后**:
- ✅ 520客户：兜底价格 = 4.55%（固定，从配置读取）
- ✅ 非520客户：兜底价格 = max(3.54 + 风险成本, 4.55)
- ✅ 计算公式保持不变：`(当前价格 - 兜底价格) × 近3个月长周期余额 / (目标价格 - 兜底价格)`

**验证示例**（520客户）:
- 当前价格: 7.99%
- 兜底价格: 4.55%
- 近3个月余额: 205万元
- 目标价格: 6.00%
- 计算: (7.99 - 4.55) × 205 / (6.00 - 4.55) = 3.44 × 205 / 1.45 = 486.55万元

**修改文件**:
- `调价计算器_PC端.html` - 修正兜底价格计算逻辑
- `调价计算器_移动端.html` - 修正兜底价格计算逻辑
- `参数配置管理_PC端.html` - 修改配置项为520客户兜底价格

使用公式手工计算：
```

  if 520客户: 4.55

对应3个月长周期目标余额 =

  (预计用款天数 / 90) × 对应3个月长周期目标余额
```
