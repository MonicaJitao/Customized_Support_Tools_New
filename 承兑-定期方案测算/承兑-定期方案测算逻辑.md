# 承兑-定期方案测算逻辑

## 页面结构

在**现有配置页面基础上新增**:
1. **测算与图表页**: 方案对比计算 + 保证金比例收益曲线图(同一页面)
2. **配置页**(已有): 只需新增承兑-定期方案相关的配置项

**注意**: Excel中的颜色仅用于字段分类识别,网页中不需要实际颜色标注

---

## 字段分类说明

为便于理解,将字段分为4类(网页中不使用颜色):

### 类别1: 默认配置字段(Excel中为灰色填充)

这些字段在配置页面中进行配置,作为计算的默认参数。

### 类别2: 用户填写字段(Excel中为黄色填充)

这些字段由用户在测算页面中填写。

### 类别3: 计算输出字段(Excel中为蓝色填充)

这些字段根据公式自动计算,只读展示。

### 类别4: 分析字段(Excel中为绿色填充)

这些字段是绝对收益分析相关的计算输出,只读展示。

---

| 字段名称 | 计算公式 | 说明 |
|---------|---------|------|

| 字段名称 | 计算公式 | 说明 |
|---------|---------|------|
| 收益金额差异 | `=B25-C25` | 方案B与方案A的收益差异 |

#### 绝对收益分析字段(类别4)

| 字段名称 | 计算公式 | 说明 |
|---------|---------|------|
| 绝对收益:在目标保证金比例不变的情况下breakeven的开票金额(万) | `目标方案(A)单票收益 / (目标方案(B)保证金比例 × 目标方案(B)保证金部分收入-支出 × 目标方案(B)开票周期/12 - (1-目标方案(B)保证金比例) × (战略业务成本-信贷ftp) × 目标方案(B)开票周期/12 - (1-目标方案(B)保证金比例) × 优化后风险成本 × 目标方案(B)开票周期/12)` | 保持保证金比例不变,达到方案(A)收益需要的开票金额 |
| 绝对收益:在目标开票金额不变的情况下breakeven的目标保证金比例 | `(12 × 目标方案(A)单票收益 + 目标方案(B)开票周期 × 目标方案(B)开票金额 × (战略业务成本-信贷ftp+优化后风险成本)) / (目标方案(B)开票周期 × 目标方案(B)开票金额 × (战略业务成本-信贷ftp+优化后风险成本+保证金部分收入-支出))` | 保持开票金额不变,达到方案(A)收益需要的保证金比例 |

### 下半部分: 保证金比例-收益曲线图

#### 图表说明

**图表类型**: 散点曲线图

**X轴**: 保证金比例 (0% - 100%, 步进1%)

**Y轴**: 收益金额 (万元)

#### 数据系列

**系列1: 目标方案(B)收益曲线**

**数据源**:
- X值: 从0到1, 步进0.01 (共101个点)
- Y值: 根据公式计算每个保证金比例对应的收益

**计算公式** (对每个保证金比例):
```javascript
收益 = (开票金额 × 保证金比例 × 保证金部分收入支出 × 开票周期 / 12
      - 开票金额 × (1 - 保证金比例) × (战略业务成本 - 信贷ftp) × 开票周期 / 12
      - 开票金额 × (1 - 保证金比例) × 优化后风险成本 × 开票周期 / 12)
      - 目标方案(A)单票收益
```

**说明**: 这条曲线展示了在不同保证金比例下,相对于方案(A)的收益变化

#### 标记点

在图表上标记以下关键点:
1. **团队长权限点**: 保证金比例=配置值时的收益
2. **分管领导权限点**: 保证金比例=配置值时的收益
3. **目标方案(B)点**: 保证金比例=用户输入值时的收益
4. **盈亏平衡点**: 收益=0时的保证金比例

## 核心计算公式详解

### 公式1: 保证金比例对应的最低贷款定价

**Excel公式**: `=ROUNDDOWN(B19*$B$9/(1-B19),2)+$B$9`

**JavaScript实现**:
```javascript
function calculateMinLoanPrice(marginRatio, creditFtp) {
  const basePrice = (marginRatio * creditFtp) / (1 - marginRatio);
  const roundedBase = Math.floor(basePrice * 100) / 100; // 向下取整到2位小数
  return roundedBase + creditFtp;
}
```

**示例**:
- 保证金比例 = 0.8
- 信贷ftp = 0.0213
- 基础价格 = (0.8 × 0.0213) / (1 - 0.8) = 0.01704 / 0.2 = 0.0852
- 向下取整 = 0.08
- 最低贷款定价 = 0.08 + 0.0213 = 0.1013 (10.13%)

### 公式2: 保证金部分收入-支出

**Excel公式**: `=IFS(B20="活期",B$5,B20="3个月定期",B$6,B20="6个月定期",B$7)`

**JavaScript实现**:
```javascript
function getMarginSpread(depositType, spreadCurrent, spread3Month, spread6Month) {
  switch(depositType) {
    case '活期': return spreadCurrent;
    case '3个月定期': return spread3Month;
    case '6个月定期': return spread6Month;
    default: return spreadCurrent;
  }
}
```

### 公式3: 业务口径单票收益

**Excel公式**: 
```
=B22*B19*B24*B21/12-B22*(1-B19)*(B8-B9)*B21/12-B22*(1-B19)*B11*B21/12
```

**公式拆解**:
```
收益 = 保证金部分收益 - 非保证金部分业务成本 - 非保证金部分风险成本

其中:
- 保证金部分收益 = 开票金额 × 保证金比例 × 保证金部分收入-支出 × 开票周期/12
- 非保证金部分业务成本 = 开票金额 × (1-保证金比例) × (战略业务成本-信贷ftp) × 开票周期/12
- 非保证金部分风险成本 = 开票金额 × (1-保证金比例) × 优化后风险成本 × 开票周期/12
```

**JavaScript实现**:
```javascript
function calculateSingleTicketRevenue(
  invoiceAmount,      // 开票金额
  marginRatio,        // 保证金比例
  marginSpread,       // 保证金部分收入-支出
  billingCycle,       // 开票周期(月)
  strategicBizCost,   // 战略业务成本
  creditFtp,          // 信贷ftp
  riskCost            // 优化后风险成本
) {
  const marginPart = invoiceAmount * marginRatio * marginSpread * billingCycle / 12;
  const nonMarginBizCost = invoiceAmount * (1 - marginRatio) * (strategicBizCost - creditFtp) * billingCycle / 12;
  const nonMarginRiskCost = invoiceAmount * (1 - marginRatio) * riskCost * billingCycle / 12;
  
  return marginPart - nonMarginBizCost - nonMarginRiskCost;
}
```

**示例计算** (目标方案B):
- 开票金额 = 2100万
- 保证金比例 = 0.8
- 保证金部分收入-支出 = 0.0212 (活期)
- 开票周期 = 6个月
- 战略业务成本 = 0.031
- 信贷ftp = 0.0213
- 优化后风险成本 = 0.0031

```
保证金部分收益 = 2100 × 0.8 × 0.0212 × 6/12 = 17.8272万
非保证金业务成本 = 2100 × 0.2 × (0.031-0.0213) × 6/12 = 2.037万
非保证金风险成本 = 2100 × 0.2 × 0.0031 × 6/12 = 0.651万

单票收益 = 17.8272 - 2.037 - 0.651 = 15.1392万
```

### 公式4: Breakeven开票金额 (保证金比例不变)

**Excel公式**: 
```
=C25/(B19*B24*B21/12-(1-B19)*(B8-B9)*B21/12-(1-B19)*B11*B21/12)
```

**JavaScript实现**:
```javascript
function calculateBreakevenInvoiceAmount(
  targetRevenue,      // 目标方案A的单票收益
  marginRatio,        // 保证金比例
  marginSpread,       // 保证金部分收入-支出
  billingCycle,       // 开票周期
  strategicBizCost,   // 战略业务成本
  creditFtp,          // 信贷ftp
  riskCost            // 优化后风险成本
) {
  const unitRevenue = marginRatio * marginSpread * billingCycle / 12 
                    - (1 - marginRatio) * (strategicBizCost - creditFtp) * billingCycle / 12 
                    - (1 - marginRatio) * riskCost * billingCycle / 12;
  
  return targetRevenue / unitRevenue;
}
```

### 公式5: Breakeven保证金比例 (开票金额不变)

**Excel公式**: 
```
=(12*C25 + B21*B22*(B8-B9+B11))/(B21*B22*(B8-B9+B11+B24))
```

**JavaScript实现**:
```javascript
function calculateBreakevenMarginRatio(
  targetRevenue,      // 目标方案A的单票收益
  invoiceAmount,      // 开票金额
  billingCycle,       // 开票周期
  strategicBizCost,   // 战略业务成本
  creditFtp,          // 信贷ftp
  riskCost,           // 优化后风险成本
  marginSpread        // 保证金部分收入-支出
) {
  const numerator = 12 * targetRevenue + billingCycle * invoiceAmount * (strategicBizCost - creditFtp + riskCost);
  const denominator = billingCycle * invoiceAmount * (strategicBizCost - creditFtp + riskCost + marginSpread);
  
  return numerator / denominator;
}
```

## 数据验证规则

### 配置页验证

1. **保证金FTP收入**: 必须 > 0, 通常在 0.01 - 0.05 之间
2. **保证金FTP活期支出**: 必须 ≥ 0
3. **保证金FTP3个月定期支出**: 必须 ≥ 0
4. **保证金FTP6个月定期支出**: 必须 ≥ 0
5. **战略业务成本**: 必须 > 0
6. **信贷ftp**: 必须 > 0
7. **优化后风险成本**: 必须 ≥ 0
8. **团队长权限**: 必须 > 0
9. **分管领导权限**: 必须 > 0

### 测算页验证

1. **目标保证金比例**: 必须在 0 - 1 之间 (0% - 100%)
2. **活期/定期**: 必须是 "活期"、"3个月定期" 或 "6个月定期"
3. **开票周期(月)**: 必须在 1 - 12 之间
4. **开票金额(万)**: 必须 > 0

## 业务场景示例

### 场景1: 活期 vs 6个月定期对比

**输入**:
- 目标方案(B): 保证金比例=80%, 活期, 开票周期=6月, 开票金额=2100万
- 目标方案(A): 保证金比例=98%, 6个月定期, 开票周期=6月, 开票金额=1000万

**配置参数**:
- 保证金FTP收入 = 2.12%
- 保证金FTP活期支出 = 0%
- 保证金FTP6个月定期支出 = 1.3%
- 战略业务成本 = 3.1%
- 信贷ftp = 2.13%
- 优化后风险成本 = 0.31%

**计算过程**:

**目标方案(B)**:
1. 保证金部分收入-支出 = 2.12% - 0% = 2.12% (活期)
2. 保证金部分收益 = 2100 × 0.8 × 0.0212 × 6/12 = 17.8272万
3. 非保证金业务成本 = 2100 × 0.2 × (0.031-0.0213) × 6/12 = 2.037万
4. 非保证金风险成本 = 2100 × 0.2 × 0.0031 × 6/12 = 0.651万
5. 单票收益 = 17.8272 - 2.037 - 0.651 = **15.1392万**

**目标方案(A)**:
1. 保证金部分收入-支出 = 2.12% - 1.3% = 0.82% (6个月定期)
2. 保证金部分收益 = 1000 × 0.98 × 0.0082 × 6/12 = 4.018万
3. 非保证金业务成本 = 1000 × 0.02 × (0.031-0.0213) × 6/12 = 0.097万
4. 非保证金风险成本 = 1000 × 0.02 × 0.0031 × 6/12 = 0.031万
5. 单票收益 = 4.018 - 0.097 - 0.031 = **3.89万**

**差异**: 15.1392 - 3.89 = **11.2492万**

## 技术实现要点

### 1. 响应式计算

当用户修改任何填写字段时,需要实时更新:
- 所有计算输出字段
- 收益金额显示
- 图表曲线

### 2. 图表绘制

使用 Chart.js 或 ECharts 绘制散点曲线:
- X轴: 0% - 100%, 步进1%
- Y轴: 根据公式计算每个点的收益值
- 标记关键点: 团队长权限、分管领导权限、目标方案点、盈亏平衡点

### 3. 数据持久化

将配置页的参数保存到 localStorage,页面刷新后自动加载

### 4. 表单验证

实时验证用户输入,不符合规则时显示错误提示
